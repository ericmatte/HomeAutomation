# Atrium Dashboard

The Atrium dashboard is a fully dynamic Lovelace dashboard generated by a custom strategy: floors, rooms, and entities are read from Home Assistant at load time, so adding an area or a light in HA is enough to update the UI — no YAML edits required.

JS sources live in [`www/atrium/`](www/atrium/):

- `strategy.js` — dashboard strategy + header + floor label (single entry point; imports the others as ES modules)
- `area-card.js` — per-floor area renderer
- `popover.js` — shared popover component

## Prerequisites

1. Install [HACS](https://hacs.xyz/) if you don't have it yet.
2. Through HACS → Frontend, install [mini-graph-card](https://github.com/kalkih/mini-graph-card) (used to render the climate background graph on each room tile).
3. Copy the `www/atrium/` folder from this repo into your Home Assistant `/config/www/atrium/` directory (file editor add-on, Samba, or `scp`).
4. One-time only, for the validation checklists shown at the top of the Routines tab: go to **Settings → Devices & services → Add integration → Local To-do**, and create two lists named exactly `Validation changement` and `Validation suivi long terme` (so they resolve to `todo.validation_changement` and `todo.validation_suivi_long_terme` — check the actual entity IDs HA assigns and adjust `LEVELS` in [`components/validation-card.js`](components/validation-card.js) if they differ). Content is then fully driven by [`validation-checklists.json`](validation-checklists.json) — see the [root CLAUDE.md](../../CLAUDE.md) for the authoring convention.

## Setup in the UI

1. Go to **Settings → Dashboards → Resources** (top-right ⋮ menu → Resources if hidden).
2. Add a new resource:
   - URL: `/local/atrium/strategy.js`
   - Resource type: **JavaScript module**
3. Reload the browser (hard refresh) so HA picks up the new resource.
4. Go to **Settings → Dashboards → Add Dashboard → New dashboard from scratch**.
   - Title: `Atrium` (or anything you like)
   - Icon: `mdi:home-variant`
   - Show in sidebar: on
5. Open the new dashboard, then **⋮ (top-right) → Edit dashboard → ⋮ → Raw configuration editor**, replace the content with:

   ```yaml
   strategy:
     type: custom:atrium
   views: []
   ```

6. Save. The dashboard rebuilds itself on every load from your HA floors/areas/entities.

## Custom tabs

`Home`, `Climate`, and `Routines` are auto-discovered from your HA floors/areas. Anything beyond that (energy monitoring, system health, etc.) is manual and config-driven via a `tabs` list on the strategy — there are zero of these until you add some, and each one you add is appended as its own tab, in order, after `Routines`:

```yaml
strategy:
  type: custom:atrium
  tabs:
    - title: Energy
      icon: mdi:lightning-bolt
      cards:
        - type: vertical-stack
          cards: [...]
    - title: Maintenance
      icon: mdi:wrench
      cards:
        - type: entities
          title: System Info
          entities: [...]
```

Per tab:

- `title` (required) — shown on the tab and as the view's header title.
- `icon` (optional) — tab icon, defaults to `mdi:view-dashboard`.
- `path` (optional) — URL path segment, defaults to `title` slugified (e.g. `Energy` → `energy`).
- `cards` (optional) — any Lovelace cards, rendered as-is below the header.
- `entities` (optional) — a plain list of entity IDs, rendered as a single `entities` card above `cards`.
- `entities_title` (optional) — title for that `entities` card; defaults to the tab's `title`.

There's no limit on how many tabs you add.
